Data Transparency — What is Real vs. Synthetic
Source data: 100% real. All 477,801 person records come directly from NHTSA CRSS public CSV archives (2020–2023). No records were fabricated or simulated as inputs to this study.
SMOTE: training set only. To address the severe class imbalance (fatal crashes = 0.9%), SMOTE (Synthetic Minority Oversampling Technique) is applied only to the training fold. It interpolates new synthetic training examples between existing real crash records. The validation set (95,894 records) and test set (94,969 records) contain only real crash records — never synthetic ones.
All reported metrics are from real data. Every F1 score, fatal recall, and accuracy figure shown in this dashboard is evaluated on the real hold-out test set exclusively.

Model Comparison

Performance Benchmark

Four models were trained on real NHTSA CRSS data: three gradient-boosted tree ensembles and one FT-Transformer. LightGBM achieves the best overall balance (54.3% fatal sensitivity, F1-macro 45.0%). The FT-Transformer reaches 57.3% fatal sensitivity from a partially-trained checkpoint (CPU-constrained training). All metrics computed on real hold-out test set.

Datasets Used

All analysis is based on public NHTSA crash data — no proprietary, simulated, or fabricated inputs. Every table below is a real government CSV file downloaded directly from NHTSA.

Primary Dataset
NHTSA Crash Report Sampling System (CRSS)
CRSS is a nationally representative probability sample of police-reported motor vehicle crashes in the United States. It uses a stratified multi-stage probability sampling design to represent the full range of crash severities — from property-damage-only through fatal. Data are collected annually by NHTSA from approximately 60 primary sampling units across the country.

From Raw Tables to 44 Model Features
The 9 tables above are joined on CASENUM / VEH_NO / PER_NO / DATA_YEAR. Supplementary tables (DISTRACT, DRIMPAIR, CRASHRF, MANEUVER, DRIVERRF, VIOLATN) are aggregated to binary flags or counts per vehicle before joining. Raw CRSS coding values (98 = Not Reported, 99 = Unknown, 998/999 = coded unknowns) are treated as missing and imputed in Step 3. Only columns that contribute to the 44 model features are loaded — unused raw columns were removed in a cleanup pass for full pipeline transparency.
